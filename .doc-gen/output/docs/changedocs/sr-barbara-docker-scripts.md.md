## File 3: `adminer-start.sh` (new file, repo root)

```bash
#!/usr/bin/env bash
# Sr. Barbara's Class — start Adminer (developer database UI)
# Requires the game stack to be running first: bash start.sh
# Visit http://localhost:8080 to use Adminer.
#
# Login:
#   Server:   db
#   Username: value of DB_USER in .env
#   Password: value of DB_PASSWORD in .env
#   Database: value of DB_NAME in .env
set -euo pipefail

if [ ! -f .env ]; then
    echo "ERROR: .env not found. Copy .env.example to .env and fill in values."
    exit 1
fi

if [ ! -f .env.dev ]; then
    echo "ERROR: .env.dev not found. Copy .env.dev.example to .env.dev and fill in values."
    exit 1
fi

# Merge env files — docker-compose on this machine only accepts one --env-file
cat .env .env.dev > .env.combined

docker-compose --env-file .env.combined \
    -f docker-compose.yaml \
    -f docker-compose.dev.yml \
    up -d

echo "Adminer running at http://localhost:8080"
```

## File 4: `adminer-stop.sh` (new file, repo root)

```bash
#!/usr/bin/env bash
# Sr. Barbara's Class — stop Adminer
set -euo pipefail

docker-compose --env-file .env.combined \
    -f docker-compose.yaml \
    -f docker-compose.dev.yml \
    stop adminer

# Clean up the combined env file
rm -f .env.combined

echo "Adminer stopped."
```

### Why adminer-stop.sh needs .env.combined

docker-compose needs to parse the compose files to know which container
to stop, and parsing `docker-compose.dev.yml` requires the variables to
be set. If `.env.combined` doesn't exist when stop is called, generate
it first:

```bash
# If you get a variable warning on stop, run this first:
cat .env .env.dev > .env.combined
bash adminer-stop.sh
```

## File 5: `docker-compose.dev.yml` (modified)

### BEFORE

```yaml
networks:
  sr_barbara_net:
    external: true
```

### AFTER

```yaml
networks:
  sr_barbara_net:
    external:
      name: sr-barbara_sr_barbara_net
```

### Why

Older docker-compose (v1) prefixes the project directory name onto network
names. The game stack creates the network as `sr-barbara_sr_barbara_net`,
not `sr_barbara_net`. The `external: name:` syntax maps the internal
reference to the actual network name, making this work regardless of
which docker-compose version is used.
