## Version stamping design

`srb-export` writes a version stamp (major.minor from `pyproject.toml`)
into the header of `sentences.yaml`:
```
# Generated by srb-export v0.1 — do not edit by hand.
```

When `build_slingcode.py` is written, it will read this stamp, compare
to the current project version, and hard-stop with a clear error message
if they don't match:
```
Error: sentences.yaml was generated with v0.1, current project is v0.2.
Run srb-export to regenerate, then retry.
```
