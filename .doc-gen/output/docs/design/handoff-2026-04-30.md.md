### 5. game.js split into six modules
All files in `static/js/`:

| File | Responsibility |
|------|---------------|
| `config.js` | All constants: canvas dims, PRAISE, SCOLD, POS_COLORS, POS_LABELS |
| `layout.js` | `estimateWidth()`, `calculateLayout()` |
| `renderer.js` | SVG helpers, `drawTokensInSection()`, `drawPhrase()` |
| `ui.js` | Legend, feedback, popup, sentence display, completion check |
| `data.js` | `fetchSentence()` — Flask API version |
| `main.js` | State, DOM refs, `phraseKey()`, `handleChoice()`, event listeners |

Load order in `index.html`: config → layout → renderer → ui → data → main.

State variables (`currentSentence`, `drawnRoles`, `ppSlot`, `layout`,
etc.) are declared inside `main.js`'s `DOMContentLoaded` closure. Other
modules access them as closed-over globals — no module system, no imports.
This is intentional and compatible with the Slingcode build.

`game.js` archived to `.archive/game.js` (gitignored).
