### 2. Architecture decisions made

**Slingcode is out.** The single-file HTML build requires no external
tool — a Python concatenation script handles everything. Slingcode was
unmaintained and unnecessary.

**Build script renamed:** `build_slingcode.py` → `build_game.py`.
The script doesn't exist yet (still a placeholder), so this is a
pending rename. Three places to update when the file is created:
- Filename: `scripts/src/sr_barbara_scripts/build_game.py`
- `pyproject.toml` entry point:
  `srb-build = "sr_barbara_scripts.build_game:main"`
- Any doc references to `build_slingcode.py`

**Game template pattern clarified:** The reusable layer across future
games (gas law game, Johnstown flood game, etc.) is the build pipeline
— not the game logic. Each game has its own renderer, content schema,
and game loop. Shared infrastructure is:
- Postgres → YAML export pattern
- `build_game.py` script (parameterized by game)
- YAML config + JSON Schema convention
- Common CSS baseline (once base.css / sr-barbara.css split is done)
